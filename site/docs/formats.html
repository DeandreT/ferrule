Chapter 4

Supported formats

Format adapters translate external documents into `ir::Instance` and write target instances back out. Extension dispatch covers common cases; embedded format options carry dialects, layouts, and schemas.

FormatSourceTargetCurrent scope
XMLYesYesHierarchical I/O, XSD-lite, bounded DTD import, attributes, `xsi:nil`, generic elements, and mixed content.
JSONYesYesHierarchical I/O, JSON Lines, local schema references, selected object alternatives, and dynamic target properties.
CSVYesYesDelimited flat rows with configurable delimiter and header handling.
Fixed-widthYesYesValidated Unicode-scalar column layouts, fill policy, record separators, and typed empty values.
XLSXYesYesTyped worksheets, selected composite/grid source shapes, hierarchical targets, and update-existing writes.
SQLiteYesYesTable introspection, typed reads, imported relational query shapes, and idempotent full-replace writes.
EDIYesYesX12, EDIFACT, HL7 v2, TRADACOMS, IDoc, and SWIFT MT through schema-guided or embedded layouts.
FlexTextYesYesRecursive split/store/switch layouts, fixed-width records, delimiter handling, and pruning rules.
Protocol BuffersYesYesBounded proto2/proto3 schemas and binary wire I/O, including nested messages, enums, and packed scalars.
XBRLYesYesTyped facts, contexts, dimensions, units, namespaces, and table-shaped runtime boundaries.
PDFYesNoLayout-driven visual extraction from text positions, anchors, painted edges, and table regions.

Boundaries

  • PDF output is intentionally unsupported.
  • XBRL taxonomy formula, presentation, calculation, and linkbase execution are not interpreted.
  • XML supports a practical schema subset rather than every namespace and `xsi:type` shape.
  • JSON Schema supports selected closed-object alternatives, not general nested union composition.
  • Database execution is SQLite-specific today.
