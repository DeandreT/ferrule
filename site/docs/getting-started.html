Getting started

The fastest path is to run the checked-in CLI fixture, then open the native editor or browser demo once you understand the project shape.

Install the toolchain

ferrule currently tracks Rust nightly.

rustup toolchain install nightly

Run an existing mapping

This example reads XML orders and writes flat CSV rows. The CLI picks input and output adapters by extension unless the project embeds more specific format options. 

cargo +nightly run -p cli -- run \
  --project crates/cli/tests/fixtures/orders/project.json \
  --input crates/cli/tests/fixtures/orders/Orders.xml \
  --output order-lines.csv

Validate without executing

Validation checks graph reachability, schema compatibility, scope controls, joins, generated sequences, and format options before any input file is read. 

cargo +nightly run -p cli -- validate \
  --project crates/cli/tests/fixtures/orders/project.json

Open the editor

The GUI is an egui desktop app. It stores canvas layout data beside the project so semantic project JSON stays readable. 

cargo +nightly run -p gui

Import schemas

Schema importers are useful when you want to create a project from existing contracts before wiring the mapping. 

cargo +nightly run -p cli -- import-xsd --xsd Orders.xsd
cargo +nightly run -p cli -- import-json-schema --schema customers.schema.json
cargo +nightly run -p cli -- import-db --db warehouse.db --table orders
Use JSON diagnostics in automation. `--diagnostics json` emits versioned JSON Lines on stderr for validation, runtime, interop, and CLI usage failures.
