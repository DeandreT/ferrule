Chapter 7

Definitions

A concise glossary for the mapping model, execution engine, format boundaries, interoperability layer, and generated-code pipeline.

Aggregate Binding Cardinality Collection Context Diagnostic Document set Dynamic field Endpoint Format adapter Frame Generated sequence Graph Group Instance IR Iteration Join Lookup Lowering MFD Node Null and XML nil Portable subset QName Scalar Schema Scope Source field Value map Window

Mapping model

Binding
A connection from one scalar graph node to a named field in the target group currently being constructed by a scope.
Cardinality
Whether a schema node or scope produces one value, an optional value, or a repeated sequence. In ferrule, repetition is explicit in the schema and instance tree.
Dynamic field
A source or target property whose name is determined at runtime. This is primarily used for open JSON objects and generic XML elements.
Graph
The set of typed computation nodes in a project. Graph nodes calculate scalar values; scopes decide where and how often those values are placed in the target.
Node
One graph operation, such as a source field, constant, function call, conditional, lookup, aggregate, runtime value, typed host parameter, or join-owned field.
Schema
A format-independent tree describing named scalar and group fields, their scalar types, repetition, attributes, nullability, and fixed values.
Scope
A target-construction unit. A scope may iterate a collection or generated sequence, apply controls such as filtering and sorting, bind scalar fields, and invoke child scopes.
Value map
An ordered table that converts one scalar value to another. The first matching row wins, with an optional explicit default when no row matches.

Runtime data

Collection
A repeated instance value addressed by a source path. Collections are the inputs to scope iteration, grouping, aggregates, lookups, and joins.
Context
The active stack of source frames visible while an expression is evaluated. It also carries iteration positions, join tuples, named inputs, fixed runtime values, and bounded typed host parameters.
Runtime parameter
A bounded named scalar supplied by the execution host rather than read from a source document. Its declaration fixes the expected scalar type; missing and invalid values remain distinct typed failures.
Execution artifact
One serialized target document produced by a successful run. A filesystem run publishes it atomically; a payload run returns its bounded bytes to the host. Both reports order primary targets first, followed by additional targets in project order.
Payload document
Bounded host-owned input bytes paired with a logical path. The path selects the format and identifies dynamic secondary inputs without granting the mapping filesystem access.
Document set
An ordered collection of typed documents that retains each member's boundary-relative path. It is used for wildcard and per-document mappings without flattening document identity.
Frame
One active source item in the execution context. A frame-pinned field selects a specific repeating ancestor instead of relying on ordinary innermost-first fallback.
Group
An ordered set of named child instances, comparable to an object, record, XML element, or database row depending on the format boundary.
Instance
The runtime data tree consumed and produced by mappings. Its principal shapes are scalar, group, repeated value, and document set.
Null and XML nil
Null means absent data. XML nil is a distinct present value originating from xsi:nil="true"; keeping them separate prevents presence information from being lost.
Scalar
One atomic value: string, signed integer, floating-point number, boolean, Null, or XML nil. Scalar functions never directly return groups or collections.

Sequence operations

Aggregate
A reduction over a collection, such as count, sum, average, minimum, maximum, join, or item-at. A computed aggregate evaluates its value expression once per item.
Generated sequence
A bounded scalar collection created by literal, length, or regular-expression tokenization, or by an inclusive integer range. It has its own item and position context.
Iteration
Repeated evaluation of a scope for each source item, generated scalar, document, or join tuple. Filtering and compaction can make output positions differ from raw source positions.
Join
A duplicate-preserving inner equijoin across two or more repeating collections. A join tuple exposes fields and raw positions from each participating source.
Lookup
A first-match scan of a collection using a key expression. A miss produces Null rather than failing the mapping.
Window
An ordered sequence limit applied after filtering, sorting, and grouping, including skip-first, first, from, inclusive range, and last.

Boundaries and interoperability

Diagnostic
A structured validation, import, generation, or runtime report that identifies the responsible node, scope, endpoint, or unsupported capability.
Endpoint
A typed input or output boundary. A project has one primary source and target and may declare additional named sources and targets.
Format adapter
A reader or writer that converts between an external representation and ferrule's instance tree. Mapping logic remains independent of this conversion.
MFD
The project-file extension used by MapForce. ferrule imports and exports a documented, clean-room interoperable subset while warning about unsupported constructs.
QName
A namespace-qualified XML name. Expanded form, such as {namespace}local, avoids ambiguity when different namespaces use the same local name.

Code generation

Intermediate representation (IR)
A representation between external formats and execution. ferrule has shared schema and instance IR plus a validated, backend-neutral program used by code generation.
Lowering
Conversion of a validated mapping project into the smaller backend-neutral code-generation program. Unsupported reachable features become capability diagnostics at this stage.
Portable subset
The mapping features with intentionally equivalent semantics in the interpreter, generated Rust, and generated C#. Unreachable unsupported nodes do not block generation.
Source field
A graph node that reads a scalar path from the current context. Unpinned reads search from the innermost frame outward, enabling parent values to broadcast into nested iterations.
Terms describe executable behavior. The architecture chapter explains how these pieces fit together, while the code-generation chapter identifies the currently portable execution surface.
