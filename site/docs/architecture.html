Chapter 3

Architecture

A ferrule project is a small mapping model over a format-independent instance tree. The interpreter and generated libraries both evaluate that model with the same scope and scalar semantics.

System map

Format adapters isolate external representations from the shared instance tree. A validated graph and scope model can then take either the interpreter path or the portable code-generation path.

Project model

  1. Schemas describe source and target trees as named scalars and groups, with repetition, attributes, nullability, fixed values, and dynamic names where target formats allow them.
  2. Graph nodes compute scalar values from source fields, constants, positions, functions, conditionals, value maps, lookups, joins, aggregates, runtime values, and XML mixed content.
  3. Scopes construct target groups by iterating source collections, generated scalar sequences, document sets, or joins, then applying filters, grouping, sorting, windows, bindings, and child scopes.
  4. Endpoints identify the primary input and output plus optional named sources and targets. Stored paths can be overridden by the CLI or host application.

Execution context

Source frames form a stack. Field resolution starts at the innermost frame and falls outward, which lets parent values broadcast into nested target rows. Repeating paths can cross several collection levels; generated sequences and joins carry their own typed item and position contexts. Missing data remains explicit as `Null` instead of aborting the run.
