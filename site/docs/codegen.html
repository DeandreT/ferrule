Chapter 6

Rust and C# code generation

ferrule lowers the portable subset of a validated project into a buildable mapping library. Rust and C# share the same neutral program so successful generation preserves evaluation order, Null behavior, output shape, and typed failures.

C#

cargo +nightly run -p cli -- generate \
  --project project.json \
  --language csharp \
  --out generated-csharp

The result is a standalone, package-free .NET 10 library. Runtime sources are vendored into the generated artifact tree, and named inputs, outputs, and bounded host parameters use typed boundary methods.

Rust

cargo +nightly run -p cli -- generate \
  --project project.json \
  --language rust \
  --out generated-rust \
  --rust-runtime-path crates/codegen-runtime

Rust generation currently links the local codegen-runtime crate by path. The generated crate exposes primary-output entry points plus output-set and named-source variants. Its execution context can borrow a validated RuntimeParameters set.

JSON application boundary

Both generated libraries can accept and return ordinary JSON text. The mapping embeds its source, named-source, primary-target, and named-target schemas, so a host does not need to manually construct ferrule instance trees for a JSON workflow.

Rust

let outputs =
  execute_json_outputs_with_sources(
    source_json,
    &[NamedJsonInput {
      name: "catalog",
      document: catalog_json,
    }],
  )?;

C#

var outputs =
  GeneratedMapping
    .ExecuteJsonOutputsWithSources(
      sourceJson,
      new[] {
        new NamedJsonInput(
          "catalog", catalogJson)
      });

Input names remain exact and duplicate-checked, while named outputs retain project order. Context-aware overloads accept the same mapping paths, stable date-time, and typed runtime parameters as the instance-tree APIs.

Bounded documents. Each JSON input and output is capped at 64 MiB and each trusted embedded schema at 1 MiB. Invalid shapes, numeric conversions, output serialization, and size excess produce typed boundary failures. These methods are deliberately JSON-only; stored format paths and options remain host metadata.

Run the examples

The repository includes one mapping project with matching Rust and C# hosts. From the repository root, generate both libraries, build both hosts, and verify both results with one command: 

./examples/codegen/run.sh

The runner reads examples/codegen/project.json, writes disposable artifacts below examples/codegen/generated/, and then executes each host against the same input. This keeps the example focused on the application boundary: pass JSON text, call the generated mapping, and handle its bounded result.

The mapping removes zero-value orders, sorts the remaining rows, assigns compact output positions, and formats invoice labels. input.json and expected-output.json show the equivalent JSON boundary values.

Generate and run Rust only

find examples/codegen/generated/rust -depth -delete 2>/dev/null || true
cargo +nightly run -p cli -- generate \
  --project examples/codegen/project.json \
  --language rust \
  --out examples/codegen/generated/rust \
  --rust-runtime-path crates/codegen-runtime

cargo +nightly run \
  --manifest-path examples/codegen/rust/Cargo.toml

Generate and run C# only

find examples/codegen/generated/csharp -depth -delete 2>/dev/null || true
cargo +nightly run -p cli -- generate \
  --project examples/codegen/project.json \
  --language csharp \
  --out examples/codegen/generated/csharp

dotnet run \
  --project examples/codegen/csharp/Ferrule.CodegenExample.csproj \
  --configuration Release
Generated trees are build artifacts. Change the shared project or either host, then rerun the script. The checked-in example owns the mapping and integration code; generation recreates the language-specific libraries from that source.

Portable subset

  • Scalar constants, ordinary and frame-pinned source fields, positions, lazy conditionals, value maps, lookups, aggregates, and a closed scalar function set.
  • Primary and static named sources, one primary target, and ordered named targets.
  • Nested source iteration, generated scalar sequences, filters, multi-key sorting, sequence windows, grouping, mapped output, recursive-filter and bounded path-hierarchy target construction, and compact positions.
  • Structured XML serialization and ordered mixed-content reconstruction with graph-computed replacements.
  • Root-context static inner joins with duplicate-preserving order and exact joined fields.
  • Typed runtime errors for arity, scalar types, missing or invalid host parameters, functions, aggregate bounds, generated sequence limits, and unsupported reachable constructs.
Generation is conservative. Unsupported reachable constructs are reported before artifacts are created. Unreachable graph nodes do not block an otherwise portable project.

Output safety

The CLI validates and stages a complete artifact tree before publishing it. Generation requires a destination that does not already exist, avoiding partial replacement of user-managed source trees.
