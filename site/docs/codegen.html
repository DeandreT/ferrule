Chapter 6

Rust and C# code generation

ferrule lowers the portable subset of a validated project into a buildable mapping library. Rust and C# share the same neutral program so successful generation preserves evaluation order, `Null` behavior, output shape, and typed failures.

C#

cargo +nightly run -p cli -- generate \
  --project project.json \
  --language csharp \
  --out generated-csharp

The result is a standalone, package-free .NET 10 library. Runtime sources are vendored into the generated artifact tree, and named inputs or outputs use typed boundary methods.

Rust

cargo +nightly run -p cli -- generate \
  --project project.json \
  --language rust \
  --out generated-rust \
  --rust-runtime-path crates/codegen-runtime

Rust generation currently links the local `codegen-runtime` crate by path. The generated crate exposes primary-output entry points plus output-set and named-source variants.

Portable subset

  • Scalar constants, ordinary and frame-pinned source fields, positions, lazy conditionals, value maps, lookups, aggregates, and a closed scalar function set.
  • Primary and static named sources, one primary target, and ordered named targets.
  • Nested source iteration, generated scalar sequences, filters, multi-key sorting, sequence windows, grouping, mapped output, and compact positions.
  • Root-context static inner joins with duplicate-preserving order and exact joined fields.
  • Typed runtime errors for arity, scalar types, functions, aggregate bounds, generated sequence limits, and unsupported reachable constructs.
Generation is conservative. Unsupported reachable constructs are reported before artifacts are created. Unreachable graph nodes do not block an otherwise portable project.

Output safety

The CLI validates and stages a complete artifact tree before publishing it. Generation requires a destination that does not already exist, avoiding partial replacement of user-managed source trees.
