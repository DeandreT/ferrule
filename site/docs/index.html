How to read this book

The early chapters cover the everyday workflow: building or importing schemas, validating a project, running a mapping, and choosing runtime formats. The later chapters explain interoperability and code generation boundaries so you can tell when a mapping will run through the interpreter, export to MapForce, or lower to generated Rust and C#. Use the definitions chapter when a mapping or runtime term is unfamiliar.

Project files are ordinary JSON. The GUI, CLI, interpreter, web demo, and code generators all work from the same model instead of translating between editor-only and runner-only project formats.

The core workflow

ferrule normalizes inputs into a shared instance tree, evaluates the mapping graph and target scopes, then writes the result through a separate format adapter. That division is what lets one mapping cross XML, JSON, tabular, database, EDI, binary, spreadsheet, and document extraction boundaries.
